06/16 使用顺序容器和分析字符串

学习要点：

  1. 容器和迭代器
  2. list 表和 vector 向量的对比
  3. 顺序容器和字符串的成员和创建方法
  4. 字符串的有用方法和cctype文件中对字符的判定

1. 容器和迭代器

在标准库中，对不同的容器，vector，list等都有着相似的操作和语义，因而有统一的操作，接下来我们就介绍这些统一操作的一个：迭代器

迭代器：

  1. 优势

    • 便于优化：作为容器的成员函数，帮助库了解我们在顺序遍历容器，基于顺序遍历库可以做出很多优化。
    • 代码优雅：替换了使用索引这样可能是用于随机存取的索引，代码更加简洁

  2. 定义：迭代器在顺序容器中都有定义，其中 iterator 可以直接转换为 const_iterator

  3. 操作：
    包含间接引用对象，取出对象元素，移动到下一位等

2. list 表和 vector 向量的对比

二者都是顺序容器，但是 vector 想要实现快速随机存取，即使用索引；而list实现在任意位置插入和删除，但不支持索引

  • vector
  1. 针对在向量末端添加和删除元素，以及根据索引随机访问进行了优化；
  2. 删除时间很慢，O(N)的删除时间，因为需要移动删除元素后面所有元素；
  3. 迭代器失效问题，需要避免对于可能无效的迭代器的复制。
    • 迭代器失效：在 student.erase(iter) 删除迭代器之后，迭代器后面所有元素都会移动位置，从而指向后面所有元素的迭代器都会失去意义
  4. 随机访问迭代器
  • list
  1. 是对删除插入任意位置元素的优化；是大规模数据任意删除元素的首选
  2. 不支持索引
  3. 不存在迭代器失效，只有删除的迭代器会失效
  4. 不支持完全随机访问，需要另外的 sort 函数
